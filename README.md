# FlowBook

## 1. ภาพรวมโครงการและคุณค่าหลัก (Project Overview & Core Value Proposition)
- **กลุ่มเป้าหมาย:** Micro-SMEs, ร้านค้าปลีกท้องถิ่น, คาเฟ่ และวิสาหกิจชุมชน
- **ปัญหา (Pain Point):** ข้อมูลทางการเงินแยกส่วนกัน ระบบ POS ทั่วไปในตลาดบันทึกได้เฉพาะฝั่งรายรับ ($Revenue$) ในขณะที่บอทไลน์หรือสมุดจดบันทึกทั่วไปรู้แค่ฝั่งรายจ่าย ($Expense$) ทำให้การกระทบยอด (Reconciliation) ตอนสิ้นเดือนมีความยุ่งยากและเกิดข้อผิดพลาดง่าย
- **แนวคิดการแก้ปัญหา:** สร้างแพลตฟอร์มบนคลาวด์ที่เป็นศูนย์กลางในการเชื่อมโยงและผสานข้อมูล (Data Pipeline) ทั้งฝั่งรายรับจากหน้าร้าน และฝั่งรายจ่ายจากบอทไลน์เข้าด้วยกันแบบอัตโนมัติ
- **คุณค่าหลัก (Core Value):** ช่วยให้เจ้าของกิจการเห็น **"กำไรสุทธิที่แท้จริง (True Net Profit)"** แบบ Real-time, ออกรายงานภาษีได้อย่างถูกต้อง และสร้างงบกำไรขาดทุน (P&L Statement) ที่เป็นระบบเพื่อใช้เป็นหลักฐานทางการเงิน (Credit Scoring) ในการยื่นขอสินเชื่อกับธนาคาร
- **รูปแบบธุรกิจ (Business Model):** วางโครงสร้างแบบ B2B2G โดยทำการตลาดกับร้านค้าในฐานะ "เครื่องมือช่วยสร้างโอกาสทางธุรกิจและเข้าถึงแหล่งเงินทุน" (ไม่เน้นชูเรื่องภาษีเพื่อลดความกังวลของร้านค้า) และเตรียมนำข้อมูลความสำเร็จ (Traction) ไปเสนอเป็นพาร์ทเนอร์เชิงกลยุทธ์กับธนาคารของรัฐ (เช่น ธนาคารออมสิน) เพื่อช่วยลดอัตราหนี้เสีย (NPL) ในการปล่อยสินเชื่อฐานราก

## 2. สถาปัตยกรรมระบบและกระบวนการทำงานที่ปรับปรุงแล้ว (Optimized Architecture & Workflow)
ระบบได้รับการอัปเกรดจากแนวคิดเดิมที่เป็นการจับคู่ไฟล์ Excel ด้วยมือ ให้กลายเป็นระบบอัตโนมัติบนคลาวด์ (Cloud-Based Automation) เพื่อรองรับการขยายสเกลและสร้างประสบการณ์การใช้งานที่ง่ายที่สุด (Frictionless UX):


```

[ ส่วนหน้าบ้าน / ส่วนรับข้อมูล ]
├── ระบบ POS หน้าร้าน (Web/Mobile App)  ───> (ส่งยอดขายผ่าน API) ───>  [ ฐานข้อมูลกลาง Cloud DB ]
└── บอทไลน์สำหรับพนักงาน (LINE Bot)      ───> (ส่ง Webhook + OCR) ───>            │
▼
[ ส่วนประมวลผลและแสดงผล ]                                              [ ระบบ Sync ข้อมูลอัตโนมัติ ]
└── แสดงผลกำไรสุทธิผ่าน หน้าเว็บ Dashboard <────────────────────────────  (จับคู่ด้วยรหัสร้าน + วันที่)

```

### รายละเอียดการทำงานของแต่ละส่วนประกอบ
| องค์ประกอบระบบ | รูปแบบเดิม (จากแบบร่างต้นแบบ) | รูปแบบที่ปรับปรุงเพื่อการผลิตจริง (Production) |
| :--- | :--- | :--- |
| **1. การเก็บข้อมูลรายรับ (POS)** | เครื่อง POS บันทึกข้อมูลยอดขาย $\rightarrow$ ดึงข้อมูลออกมาเป็นไฟล์ Excel แมนนวล | **ระบบ POS แบบ Web/Mobile App** ที่ส่งข้อมูลธุรกรรมดิบ (รายการ, จำนวน, ราคา, เวลา, รหัสลูกค้า) ตรงเข้าสู่ Cloud DB ทันทีผ่าน RESTful API หลังปิดการขาย |
| **2. การเก็บข้อมูลรายจ่าย (LINE Bot)** | พิมพ์ข้อความ, ส่งภาพใบเสร็จ หรือไฟล์เข้า LINE Bot $\rightarrow$ แปลงข้อมูลเป็นไฟล์ Excel รายจ่าย | **LINE Messaging API + ระบบ AI/OCR Parsing**. เมื่อผู้ใช้ส่งรูปใบเสร็จหรือพิมพ์ (เช่น "ค่าน้ำแข็ง 50 บาท") ระบบ Webhook จะสกัดข้อความ ดึงยอดเงิน จัดหมวดหมู่ และบันทึกลง Cloud DB ทันที |
| **3. การรวมข้อมูล (Harmonization)** | นำไฟล์ Excel รายรับ และ Excel รายจ่าย มารวมกันภายหลังด้วยมือหรือสูตรคอมพิวเตอร์ | **ระบบฐานข้อมูลความสัมพันธ์ (Relational Database)** จับคู่และเชื่อมโยงข้อมูลรายรับ-รายจ่ายเข้าด้วยกันทันทีแบบอัตโนมัติ โดยใช้ Composite Primary Key: `[Shop_ID + Date]` |
| **4. การประมวลผลลัพธ์สุดท้าย** | ได้ไฟล์รวม รายรับ + รายจ่าย 100% สำหรับนำไปคำนวณภาษีต่อ | **หน้าเว็บ Dashboard กลางแบบไดนามิก**. ประมวลผลกราฟการเงินของร้าน แสดงงบกำไรขาดทุน Real-time และกดส่งออก (Export) เป็นเอกสารรายงานภาษีหรือเอกสารยื่นกู้ธนาคารได้ในคลิกเดียว |

## 3. แผนการดำเนินงานเชิงกลยุทธ์ (Strategic Execution Roadmap)
เพื่อรักษาอำนาจการตัดสินใจและสิทธิ์ในทรัพย์สินทางปัญญา (IP) ของทีมอิสระ (Cross-University Team) ไว้ 100% แผนงานจะถูกแบ่งออกเป็น 3 ระยะ:

### ระยะที่ 1: พัฒนาตัวต้นแบบและทดสอบตลาด (MVP & Traction Validation)
- **เป้าหมายทางเทคนิค:** พัฒนาตัวผลิตภัณฑ์ขั้นต่ำ (MVP) ที่มีระบบ API เส้นหลักของ POS และระบบคำหลัก (Keyword/NLP) พื้นฐานของ LINE Bot เพื่อเชื่อมต่อคลาวด์
- **การทดลองภาคสนาม:** นำระบบเวอร์ชันแรกไปให้ร้านค้าในพื้นที่จริง (เช่น คาเฟ่ หรือร้านอาหารขนาดเล็ก) ทดลองใช้งานฟรี 3-5 ร้าน เป็นเวลา 30 วัน
- **ตัวชี้วัดที่ต้องเก็บข้อมูล:** อัตราการใช้งานต่อเนื่องของผู้ใช้ (User Retention Rate) และความแม่นยำของระบบถอดรหัสรายจ่ายผ่าน LINE

### ระยะที่ 2: จัดตั้งบริษัทและวางโครงสร้างกฎหมาย (Corporate Formation & PDPA)
- **การจดทะเบียนนิติบุคคล:** จัดตั้ง **"บริษัทจำกัด (Thai Limited Company)"** ร่วมกันเพื่อแบ่งสัดส่วนหุ้นและสิทธิ์ในตัวโปรดักต์ระหว่างสมาชิกต่างมหาวิทยาลัยให้ชัดเจนและถูกต้องตามกฎหมาย
- **ธรรมาภิบาลข้อมูล:** ออกแบบระบบความปลอดภัยและการจัดเก็บข้อมูลทางการเงินของร้านค้าให้สอดคล้องกับ พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (**PDPA**) พร้อมทั้งจัดทำเงื่อนไขการให้บริการ (Terms of Service) ที่รัดกุม

### ระยะที่ 3: เปิดตัวเชิงพาณิชย์และเจรจาพาร์ทเนอร์ (Commercial Launch & GSB Partnership)
- **กลยุทธ์เข้าสู่ตลาด (GTM):** ทำการตลาดและขายตรงสู่กลุ่ม SME เน้นชูจุดขายเรื่อง "การเช็กสุขภาพทางการเงินและกำไรของร้านได้ทันที" เพื่อสร้างฐานลูกค้าแบบออร์แกนิก
- **การพิตช์แผนธุรกิจ B2B2G:** นำบริษัทและข้อมูลผู้ใช้งานจริง (Traction Data) จากเฟสแรก เข้าพบฝ่ายสินเชื่อและทุนสนับสนุน Startup ของธนาคารออมสิน (GSB) นำเสนอแพลตฟอร์มในฐานะเครื่องมือช่วยคัดกรองความเสี่ยงในการปล่อยกู้ให้กลุ่มผู้ประกอบการฐานราก เพื่อสร้างข้อตกลงร่วม (MOU) ในการขยายสเกลระบบร่วมกัน
